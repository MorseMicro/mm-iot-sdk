Morse Micro IoT SDK  2.8.2
Introduction

The Morse Micro IoT SDK provides a framework for developing embedded applications that include connectivity provided by Morse Micro MM610x transceivers.

This SDK targets the Morse Micro IoT Reference Platforms, which is made up of an MCU host development board and a Morse Micro MMECH08 development board, as shown in the diagram below. The two main components are the Morse Micro MM610x Wi-Fi HaLow transceiver and an MCU Host for running application firmware. See Morse Micro IoT Reference Platforms for a full list of supported platforms.

Morse Micro IoT Reference Platform Overview

This SDK package contains the software components to build the application firmware for the STM32 processor. These components include the RTOS (FreeRTOS), network stack (LwIP), Morse Micro host software (Morselib), and the platform-specific board support package (BSP). A block diagram of the SDK software components is shown below.

High-level block diagram of the SDK software components

The Morse Micro host software included in Morselib provides the driver interface for communicating with the transceiver and the upper layers of the Wireless LAN MAC, as well as additional application-layer network protocols. APIs provided by Morselib include:

  • A WLAN Control API that can be used by the application code to manage Wireless LAN operations such as connecting to an Access Point.
  • Morse Micro Embedded Test Engine (Emmet) API for test automation.
  • A WLAN Datapath API, Morse Micro Operating System Abstraction Layer (mmosal) API, and Morse Micro Hardware Abstraction Layer (mmhal) API, which provide abstract interfaces to the IP stack, RTOS, and MCU BSP respectively. Shim layers provide the mapping between these Morselib APIs and the corresonding LwIP/FreeRTOS/BSP API.

In addition to the APIs provided by Morselib, supporting infrastructure includes:

  • Morse Micro IP Stack Abstraction Layer (MMIPAL) API, which provides a layer of abstraction to simplify management of network stacks, with support included for Lightweight IP (LWIP) and FreeRTOS-plus-TCP network stacks.
  • Application Layer Protocol APIs for Morse Micro implementations of protocols including ping and iperf.
  • Morse Micro Persistent Configuration Store APIs for storing configuration data, etc.

The Getting Started Guide provides step-by-step instructions for getting up and running with this SDK using a Makefile-based workflow.

System Requirements

Core architecture

Supported architectures:

  • ARM Cortex-M4F
  • ARM Cortex-M33

In addition, untested Morselib build(s) are provided for the following architectures:

  • ARM Cortex-M7F
  • ARM fa626te

RAM and flash

Minimum* Recommended
RAM 128 KB 192 KB+
Flash 768 KB 2 MB+

* Note that Minimum is that bare minimum required to build a simple application. This may not be enough to support, for example, cloud connectivity where application layer protocols such as TLS and MQTT are required.

Ping and iperf example applications for the reference platforms require approx 155 KB RAM as built with the out-of-the-box configuration. For low throughput applications heap and lwIP pooled memory can be tuned to reduce memory usage.

Ping and iperf example applications for the reference platforms require approx 715 KB flash as built with the out-of-the-box configuration (excluding configuration store).
